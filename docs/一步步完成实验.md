# 一步步完成实验

## 第 1 步：跑通基础版本
1. 双击 `code/run_all.bat`
2. 确认终端出现 `Correctness check passed.`
3. 打开 `code/results/`，确认 4 个 CSV 已生成

## 第 2 步：采集平台信息
1. 运行 `code/collect_system_info.ps1`
2. 把生成的 `system_info.txt` 内容复制到报告的实验平台部分
3. 你还可以手动补充 CPU cache 大小（可在任务管理器或 CPU-Z 中查看）

## 第 3 步：补做进阶要求
### A. 编译器优化力度对比
运行 `code/run_flags_compare.bat`
然后比较：
- `dot_basic_O0.csv` 与 `dot_basic_O2.csv` 与 `dot_basic_O3.csv`
- `sum_advanced_O0.csv` 与 `sum_advanced_O2.csv` 与 `sum_advanced_O3.csv`

### B. 循环展开
你已经有 `dot_cache_unroll4` 和 `sum_dual_unroll4`，只需要把它们与基础版本比较并写入报告。

### C. Godbolt / 汇编分析（可选但很推荐）
1. 打开 `https://godbolt.org`
2. 复制 `sum_naive` 与 `sum_dual_unroll4` 的函数代码
3. 分别查看 `-O0/-O2/-O3` 下生成的汇编
4. 截图贴到报告“进阶要求”部分

### D. VTune / perf（可选）
- Windows 原生更适合 VTune
- 如果以后配置 WSL/Linux，再做 perf
- 本包附带了框架，但不要求你现在必须完成这一步

## 第 4 步：把图补进报告
你现在已经有表格数据了，图可以稍后画。LaTeX 模板里已经预留了图的位置，替换占位框即可。

## 第 5 步：最后交付前检查
- 基础要求 4 项是否都完成
- 进阶要求至少完成 1-2 项（建议：循环展开 + 编译器优化对比）
- 报告是否写清平台、算法、测试方法、结果、分析
- 是否给出源码仓库链接
